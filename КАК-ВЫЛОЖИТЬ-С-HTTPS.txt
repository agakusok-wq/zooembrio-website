  ZOOEMBRIO — как выложить сайт в интернет с HTTPS (защищённым)
═══════════════════════════════════════════════════════════════

ПОЧЕМУ НУЖЕН HTTPS
──────────────────
• Браузеры помечают http-сайты как «небезопасные»
• Антивирусы и корпоративные сети чаще блокируют сайты без SSL
• Форма обратной связи и доверие клиентов работают только через https://

Локально (http://localhost:4321) HTTPS не нужен — это только для просмотра у себя на компьютере.


РЕКОМЕНДУЕМЫЙ СПОСОБ: Cloudflare Pages (бесплатный HTTPS)
──────────────────────────────────────────────────────────

Шаг 1. Аккаунты
   • GitHub:  https://github.com
   • Cloudflare: https://dash.cloudflare.com/sign-up

Шаг 2. Загрузить проект на GitHub
   • Создайте репозиторий (например zooembrio-website)
   • Загрузите папку проекта (можно через GitHub Desktop или git push)

Шаг 3. Подключить Cloudflare Pages
   • Cloudflare → Workers & Pages → Create → Pages → Connect to Git
   • Выберите репозиторий
   • Настройки сборки:
       Build command:    npm run build
       Build output:     dist
       Node version:     20 (в Environment variables или в настройках)
   • Save and Deploy

Шаг 4. Привязать домен zooembrio.ru
   • В Cloudflare Pages → ваш проект → Custom domains → Set up a domain
   • Введите: zooembrio.ru
   • Cloudflare покажет, какие DNS-записи нужны

Шаг 5. DNS у регистратора домена
   • Зайдите туда, где покупали zooembrio.ru
   • Укажите DNS-серверы Cloudflare (их даст Cloudflare при добавлении сайта)
     Обычно это что-то вроде:
       ada.ns.cloudflare.com
       bob.ns.cloudflare.com

Шаг 6. Включить принудительный HTTPS
   • Cloudflare → ваш сайт → SSL/TLS → Overview
       Режим: Full (или Full strict после выпуска сертификата)
   • SSL/TLS → Edge Certificates
       Включите: Always Use HTTPS  → ON
       Включите: Automatic HTTPS Rewrites → ON

Шаг 7. Проверка
   • Откройте https://zooembrio.ru
   • В адресной строке должен быть замочек
   • http://zooembrio.ru должен автоматически перенаправлять на https://


ЧТО УЖЕ НАСТРОЕНО В ПРОЕКТЕ
──────────────────────────
• astro.config.mjs — адрес сайта https://zooembrio.ru
• public/_headers — заголовки безопасности (HSTS, защита от подмены)
• Редиректы http/www настраиваются в панели Cloudflare (Rules → Redirect Rules)
• Форма контактов отправляется на https://formsubmit.co (защищённо)
• Шрифты Google загружаются по HTTPS


АЛЬТЕРНАТИВА: Timeweb / Beget / другой хостинг
──────────────────────────────────────────────
1. Соберите сайт: npm run build
2. Загрузите ВСЁ содержимое папки dist/ на хостинг (FTP или файловый менеджер)
3. В панели хостинга привяжите домен zooembrio.ru
4. Включите бесплатный SSL-сертификат Let's Encrypt (кнопка в панели)
5. Включите «Перенаправление HTTP → HTTPS» в настройках домена
6. Скопируйте файл _headers из public/ в корень сайта на хостинге
   (он уже попадает в dist/ при сборке Astro)


ЕСЛИ АНТИВИРУС ВСЁ ЕЩЁ РУГАЕТСЯ
───────────────────────────────
• Убедитесь, что открываете именно https://zooembrio.ru (не http)
• Подождите 24–48 часов после первого деплоя — новым доменам нужно время
• Проверьте сертификат: https://www.ssllabs.com/ssltest/ (введите zooembrio.ru)
• Не используйте самоподписанные сертификаты — только Let's Encrypt или Cloudflare
